Machine Learning — Interview Questions

Basics 20 questions
Fundamental concepts and definitions
1
What is machine learning?
Algorithms that learn patterns from data to make predictions or decisions automatically.
2
Difference between AI, ML and Deep Learning?
AI is the broad field; ML uses data to learn; DL uses deep neural networks within ML.
3
What are main types of machine learning?
Supervised, Unsupervised, Semi-supervised, Reinforcement learning.
4
What is supervised learning?
Training models on labeled examples to map inputs to outputs.
5
What is unsupervised learning?
Discovering structure in unlabeled data, e.g., clustering or dimensionality reduction.
6
What is reinforcement learning?
Learning via interactions and rewards to maximize cumulative return.
7
What is feature engineering?
Creating and transforming inputs (features) to improve model performance.
8
What is model evaluation?
Measuring performance using metrics on validation/test data to judge generalization.
9
Difference between train, validation, and test sets?
Train fits model; validation tunes hyperparameters; test reports final performance.
10
What is cross-validation?
Technique to evaluate stability by splitting data into folds (e.g., k-fold CV).
11
What are bias and variance?
Bias = error from wrong assumptions; variance = error from sensitivity to data fluctuations.
12
What is overfitting?
Model learns noise and performs poorly on unseen data.
13
What is underfitting?
Model too simple to capture underlying patterns — poor train and test performance.
14
What is regularization?
Adding penalty terms (L1/L2) to reduce overfitting by constraining weights.
15
What is a hyperparameter?
A parameter set before training (e.g., learning rate, number of trees) tuned externally.
16
Common supervised learning problem types?
Regression (continuous target) and classification (discrete labels).
17
Common unsupervised tasks?
Clustering, anomaly detection, dimensionality reduction, density estimation.
18
What is an ML pipeline?
Sequence: data ingestion → preprocessing → training → evaluation → deployment.
19
How to ensure reproducibility?
Fix seeds, record environment, store splits and checkpoints, document experiments.
20
What is model explainability?
Understanding and communicating why a model makes particular predictions.
Supervised Learning 25 questions
Algorithms, trees, SVM, regression and classification
21
What is linear regression?
Modeling a continuous target as a linear combination of features.
22
What is logistic regression?
A classifier modeling probability using the logistic (sigmoid) function.
23
What is a decision tree?
A flowchart-like model using feature splits to make predictions.
24
Gini vs entropy in trees?
Both measure impurity; Gini slightly faster, entropy uses information gain.
25
What is tree pruning?
Removing branches to reduce overfitting (pre or post pruning).
26
What is Random Forest?
Ensemble of trees on bootstrapped samples with feature randomness to reduce variance.
27
Bagging vs Boosting?
Bagging parallelizes to reduce variance; boosting sequentially reduces bias.
28
What is SVM and kernel trick?
SVM finds max-margin hyperplane; kernels map inputs to higher-dim spaces implicitly.
29
What is KNN?
Lazy classifier predicting by majority label of nearest neighbors using a distance metric.
30
What is Naive Bayes?
Probabilistic classifier assuming conditional independence between features.
31
What is stacking (stacked generalization)?
Combine base models and train a meta-learner on their outputs to improve predictions.
32
Why are XGBoost/LightGBM popular for tabular data?
Highly optimized gradient boosting implementations with fast training and good defaults.
33
What is probability calibration?
Adjusting predicted probabilities to match observed frequencies (e.g., Platt scaling).
34
How handle class imbalance?
Resampling, class weights, threshold tuning, or synthetic oversampling (SMOTE).
35
How to solve multiclass with binary classifiers?
One-vs-rest or one-vs-one strategies, or use multinomial classifiers directly.
36
What is ordinal regression?
Predicting ordered categories while preserving rank information between classes.
37
Common regression metrics?
MSE, RMSE, MAE, R-squared, and mean absolute percentage error (MAPE).
38
Common classification metrics?
Accuracy, precision, recall, F1-score, ROC-AUC, PR-AUC.
39
Probabilistic vs deterministic models?
Probabilistic output probabilities; deterministic return specific predictions without uncertainty.
40
What does regression intercept mean?
Baseline prediction value when all features are zero (context-dependent).
41
What is a residual?
Difference between observed and predicted target for each example.
42
What is heteroscedasticity?
Non-constant variance of residuals across different levels of a variable.
43
Why regularize linear models?
To prevent overfitting, reduce variance, and handle multicollinearity.
44
What is multicollinearity?
High correlation between features causing unstable coefficient estimates.
45
When is feature scaling needed?
Required for distance-based and gradient-based algorithms (KNN, SVM, NN), less for trees.
Unsupervised Learning 20 questions
Clustering, dimensionality reduction, density methods
46
What is clustering?
Grouping similar data points into clusters without labels.
47
Explain K-means clustering.
Iterative algorithm assigning points to nearest centroid and updating centroids.
48
What is hierarchical clustering?
Builds nested clusters using agglomerative or divisive approaches forming a dendrogram.
49
What is DBSCAN?
Density-based clustering algorithm that finds arbitrarily shaped clusters and noise.
50
What is silhouette score?
Metric measuring cohesion and separation of clusters (range -1 to 1).
51
What is PCA?
Linear dimensionality reduction projecting data onto orthogonal components of max variance.
52
What is t-SNE?
Non-linear visualization technique preserving local structure for high-dim data.
53
What is UMAP?
Manifold learning technique for fast, structure-preserving low-dimensional embeddings.
54
Why use dimensionality reduction?
Reduce noise, visualization, speed up models, mitigate curse of dimensionality.
55
What is anomaly detection?
Identifying rare or unusual observations differing from normal patterns.
56
What are Gaussian Mixture Models?
Probabilistic models representing data as mixture of Gaussian distributions with EM algorithm.
57
What is biclustering?
Simultaneous clustering of rows and columns to find submatrix patterns.
58
What is spectral clustering?
Uses graph Laplacian eigenvectors to partition data using connectivity information.
59
How choose k in K-means?
Use elbow method, silhouette score, or domain knowledge to select k.
60
What is unsupervised feature extraction?
Learning representations from unlabeled data for downstream tasks (e.g., autoencoders).
61
What is topic modeling?
Unsupervised technique (LDA) to discover topics in text collections.
62
Pitfalls of dimensionality reduction?
Loss of interpretability, information loss, and misleading visualizations if misused.
63
What is a denoising autoencoder?
Autoencoder trained to reconstruct clean inputs from noisy versions for robust features.
64
What are latent variable models?
Models that express observations via underlying unobserved variables (e.g., PCA, factor analysis).
65
What is density estimation?
Estimating probability distribution of data (parametric or non-parametric like KDE).
Feature Engineering 20 questions
Feature creation, selection, encodings
66
Why is feature engineering important?
Good features often matter more than model choice for performance gains.
67
What is one-hot encoding?
Convert categorical values into binary indicator vectors for each category.
68
When use label encoding?
Map categories to integers; be careful with ordinal implications for models.
69
What is target (mean) encoding?
Replace categories with aggregated target statistics (use smoothing to avoid leakage).
70
How handle missing values?
Imputation (mean/median/mode), model-based, or special marker values depending on context.
71
How detect and handle outliers?
Use IQR, z-score, or model-based methods; treat, cap, or remove as appropriate.
72
Scaling: standard vs min-max vs robust?
Standardize for z-scores, min-max for bounded ranges, robust uses medians for outliers.
73
What are polynomial features?
Create interaction and power terms to model non-linearities with linear models.
74
Feature selection methods?
Filters, wrappers, and embedded methods (e.g., LASSO, tree importance, recursive feature elimination).
75
When to use dimensionality reduction?
When many correlated features, for visualization, noise reduction, or speed improvements.
76
Feature ideas for time series?
Lag features, rolling stats, seasonality flags, time-based decompositions.
77
Feature extraction for text?
TF-IDF, word embeddings, n-grams, topic features, sequence encodings.
78
Feature extraction for images?
Convolutional features, pre-trained CNN embeddings, color and texture statistics.
79
What are feature interactions?
Combinations of features whose joint values better explain the target than individually.
80
Handling rare categories?
Group rare levels, use target smoothing, or frequency encoding.
81
What is data leakage?
When training data contains future or target-related information not available at inference time.
82
How split time-series data?
Use chronological splits (train earlier periods, test later) to avoid leakage.
83
Problems with high-cardinality categorical features?
Feature explosion with one-hot; prefer embeddings, target/ frequency encoding, hashing.
84
When missingness is informative?
Missing values carry signal; add an indicator feature to capture that information.
85
What is automated feature engineering?
Tools like Featuretools that generate features via domain primitives and aggregations.
Model Evaluation & Metrics 25 questions
Metrics, curves, calibration & validation
86
What is a confusion matrix?
Table showing counts of true vs predicted classes (TP, FP, TN, FN).
87
Precision vs recall?
Precision measures correctness of positives; recall measures coverage of actual positives.
88
What is F1-score?
Harmonic mean of precision and recall, balances both metrics.
89
What is ROC-AUC?
Area under ROC curve; measures separability across thresholds (0.5 random, 1 perfect).
90
When use Precision-Recall curve?
Useful with imbalanced datasets focusing on positive-class performance.
91
Limitations of AUC?
Insensitive to calibration and may hide threshold-specific performance differences.
92
What is Brier score?
Mean squared error between predicted probabilities and actual outcomes (lower is better).
93
What is log loss (cross-entropy)?
Loss measuring difference between predicted probabilities and true labels; penalizes confident wrong predictions.
94
What is a calibration curve?
Plot comparing predicted probabilities with observed frequencies to check calibration.
95
What is KS statistic?
Measure used in binary classification to compare cumulative distributions of scores between classes.
96
Types of cross-validation?
k-fold, stratified k-fold, time-series split, leave-one-out, repeated CV.
97
Holdout vs cross-validation?
Holdout faster but higher variance; CV provides more robust performance estimates.
98
How detect data leakage?
Unusually high validation performance, features correlated with target from future info.
99
How compare models statistically?
Use paired tests (e.g., McNemar, paired t-test on metrics via resampling) with caution.
100
Why compute confidence intervals for metrics?
To quantify uncertainty around performance estimates due to finite samples.
101
When use cost-sensitive evaluation?
When false positives/negatives have different real-world costs; customize loss accordingly.
102
How align ML metrics with business KPIs?
Translate model outputs to business impact, choose metrics reflecting product goals.
103
How use early stopping?
Monitor validation metric and stop when performance stops improving (with patience setting).
104
Why use stratified sampling?
To preserve class proportions across splits and get stable metric estimates for imbalanced data.
105
What are weighted metrics?
Aggregate metrics using class weights to reflect importance or imbalance.
106
What is perplexity (NLP)?
Exponentiated cross-entropy indicating how well a language model predicts a sample.
107
When prefer MAE over MSE?
MAE is robust to outliers; MSE penalizes large errors more strongly.
108
How evaluate multilabel classification?
Use micro/macro averaged precision, recall, F1 and subset accuracy depending on needs.
109
Common ranking metrics?
MAP, NDCG, precision@k, recall@k used for recommendation and retrieval tasks.
110
How use bootstrap for CI?
Resample test set many times, compute metric distribution and derive percentiles for intervals.
Statistics for ML 20 questions
Probability, inference, tests
111
What is probability?
Quantifies uncertainty of events; range 0–1 based on outcomes and assumptions.
112
State Bayes' theorem.
P(A|B)=P(B|A)P(A)/P(B): update beliefs with evidence and prior.
113
What is the Central Limit Theorem?
Sample means approximate normal distribution as sample size grows, regardless of original distribution.
114
What is hypothesis testing?
Framework to test claims using null/alternative hypotheses and p-values.
115
What is a p-value?
Probability of observing data as extreme under the null hypothesis (not the probability null is true).
116
What is a confidence interval?
Range that likely contains parameter value with a given confidence level (e.g., 95%).
117
Type I vs Type II error?
Type I: false positive (reject true null). Type II: false negative (fail to reject false null).
118
What is statistical power?
Probability to detect an effect when it exists (1 - Type II error rate).
119
What is estimator bias?
Difference between expected estimate and true parameter value.
120
Why use unbiased estimators?
To ensure expected estimate equals true parameter, but may sacrifice variance.
121
Correlation vs causation?
Correlation measures association; causation implies one variable changes another (needs experiments/careful design).
122
Pearson vs Spearman correlation?
Pearson measures linear correlation; Spearman ranks to measure monotonic relationships.
123
When use Chi-square test?
Test independence for categorical variables in contingency tables.
124
T-test vs ANOVA?
T-test compares two means; ANOVA compares means across multiple groups.
125
What is Bayesian inference?
Update posterior beliefs by combining priors with likelihood from data.
126
What is likelihood?
Function of parameters given observed data used for inference and MLE.
127
What is maximum likelihood estimation (MLE)?
Choose parameters that maximize the likelihood of the observed data.
128
Bayesian vs Frequentist?
Bayesian uses priors and yields distributions; frequentist relies on long-run frequencies and point estimates.
129
Common sampling methods?
Random, stratified, cluster, systematic sampling depending on study design.
130
Confidence vs credible interval?
Confidence intervals: frequentist interpretation; credible intervals: Bayesian probability for parameter given data.
Optimization & Training 20 questions
Gradient descent, optimizers, batch, learning rates
131
What is gradient descent?
Iterative method to minimize loss by stepping in negative gradient direction.
132
SGD vs batch vs mini-batch?
Batch uses whole data, SGD single examples, mini-batch balances stability and speed.
133
Why is learning rate important?
Controls step size; too large diverges, too small slows convergence.
134
What is momentum in optimization?
Accumulates past gradients to accelerate and smooth updates, escaping shallow minima.
135
What is Adam optimizer?
Adaptive optimizer combining momentum and RMSprop ideas with per-parameter learning rates.
136
What are learning rate schedules?
Rules to decrease LR over time: step decay, exponential, cosine annealing, cyclic schedules.
137
What is gradient clipping?
Limit gradient norm to prevent exploding gradients in deep networks.
138
Why care about weight initialization?
Proper init (Xavier/He) avoids vanishing/exploding gradients and speeds convergence.
139
What is batch normalization?
Normalize layer inputs over mini-batch to stabilize and accelerate training.
140
What is early stopping?
Stop training when validation metric stops improving to prevent overfitting.
141
What is dropout?
Randomly drop neurons during training to reduce co-adaptation and overfitting.
142
What is label smoothing?
Replace one-hot targets with softened distributions to improve generalization and calibration.
143
Why use data augmentation?
Increase training data variability to reduce overfitting and improve robustness.
144
What is mixed precision training?
Use lower-precision (float16) where safe to speed up training and reduce memory.
145
What is SWA (stochastic weight averaging)?
Average weights from multiple checkpoints to get a flatter minima and better generalization.
146
How balance batches for imbalanced labels?
Use stratified or class-balanced sampling to ensure each batch contains representative classes.
147
How choose optimizer?
Start with Adam for quick convergence; SGD with momentum often generalizes better for some tasks.
148
What is curriculum learning?
Train models on easier examples first, gradually increasing difficulty to improve learning.
149
Why inject noise during training?
Acts as regularization, improves robustness and generalization (e.g., Gaussian noise on inputs).
150
What are second-order optimization methods?
Use curvature (Hessian) info (e.g., Newton) for faster convergence but higher compute cost.
Deep Learning 25 questions
Neural nets, CNNs, RNNs, attention, transformers
151
What is an artificial neural network?
Computational model of interconnected neurons learning representations via layers and activations.
152
Common activation functions?
ReLU, sigmoid, tanh, softmax, Leaky ReLU, GELU.
153
What is the vanishing gradient problem?
Gradients shrink in deep nets (esp. sigmoid/tanh), hindering learning in early layers.
154
What are key CNN components?
Convolution, activation, pooling, normalization, fully-connected and skip connections.
155
RNN vs LSTM vs GRU?
LSTM/GRU use gates to handle long-term dependencies; GRU is simpler with comparable performance.
156
What is attention?
Mechanism to weight importance of input elements when producing outputs, enabling direct context access.
157
What are Transformers?
Architectures using self-attention and feed-forward layers for parallel sequence modeling (BERT/GPT).
158
What is transfer learning in DL?
Fine-tune pre-trained models or use embeddings to save data and compute.
159
Why use batch normalization?
Stabilizes distributions, speeds training, allows higher learning rates, and regularizes a bit.
160
What are skip/residual connections?
Shortcuts that add inputs to outputs improving gradient flow and enabling deeper nets.
161
Attention vs convolution?
Attention models global relationships; convolution captures local patterns via shared filters.
162
Common loss functions in DL?
Cross-entropy, MSE, hinge loss, CTC loss, contrastive losses depending on task.
163
What is gradient checkpointing?
Save memory by recomputing intermediate activations during backward pass at cost of extra compute.
164
How model sequences?
Use RNNs, CNNs for sequences, or Transformers for parallel context-aware sequence modeling.
165
What are word embeddings?
Dense vector representations capturing semantic similarity among words (Word2Vec, GloVe, contextualized ones).
166
Are attention weights interpretable?
Sometimes informative but not strictly explanations; use caution and complementary explainability methods.
167
What is seq2seq modeling?
Encoder-decoder architectures mapping input sequences to output sequences (translation, summarization).
168
What is beam search?
Heuristic search keeping top-k hypotheses during sequence generation to balance breadth and depth.
169
Why positional encodings in Transformers?
Provide sequence order information since attention is permutation-invariant.
170
Fine-tuning best practices?
Lower LR, gradual unfreezing, use regularization and validation to avoid catastrophic forgetting.
171
What is self-supervised learning?
Train on proxy tasks using unlabeled data to learn useful representations (e.g., contrastive methods).
172
What is contrastive learning?
Learn representations by pulling similar pairs together and pushing dissimilar pairs apart.
173
What is knowledge distillation?
Train a smaller student model to mimic a larger teacher’s outputs for efficient inference.
174
What is catastrophic forgetting?
Model loses earlier learned knowledge when trained sequentially on new tasks.
175
DL-specific evaluation challenges?
Large compute variability, randomness, dataset biases, and reproducibility concerns.
Production & MLOps 25 questions
Deployment, monitoring, drift, serving, scaling
176
How to serialize and save models?
Use framework serialization (pickle, joblib, ONNX, TorchScript) and store versioned artifacts.
177
Common model serving options?
REST/gRPC APIs, batch jobs, streaming, serverless endpoints, and specialized serving systems.
178
Why containerize models?
Provides reproducible runtime, dependency isolation, and easy deployment across environments.
179
What to monitor in production?
Performance, latency, error rates, data drift, prediction distributions, and resource usage.
180
What is data drift?
Change in input distribution over time degrading model performance.
181
What is concept drift?
Change in relationship between inputs and target, e.g., user behavior shifts affecting labels.
182
Retraining strategies for drift?
Periodic retraining, triggered retraining, online updates, or active learning pipelines.
183
What is a canary deployment?
Gradually roll out new model to a subset of traffic to monitor before full release.
184
What is shadow testing?
Run a new model in parallel without affecting live responses to compare behavior on real traffic.
185
What is a feature store?
Centralized system to store, serve, and manage features consistently for training and serving.
186
Why version models and data?
Traceability, reproducibility, rollback capability, and compliance needs.
187
How reduce inference latency?
Quantization, pruning, batching, model distillation, caching, and hardware acceleration.
188
How scale model throughput?
Horizontal scaling, autoscaling, efficient batching, sharding, and async processing.
189
Why need explainability in production?
For trust, debugging, compliance, and to communicate decisions to stakeholders.
190
Privacy-preserving ML approaches?
Federated learning, differential privacy, secure multi-party computation.
191
How manage model serving costs?
Use efficient models, autoscaling, spot instances, batching, and monitoring to optimize usage.
192
How run A/B tests with models?
Randomize users to variants and measure business metrics for significance and impact.
193
Rollback strategies for bad models?
Keep previous stable versions, use feature flags and immediate rollback triggers in pipelines.
194
Useful monitoring alerts?
Sudden metric drops, data distribution shifts, error spikes, and latency increases.
195
Regulatory concerns in ML systems?
Fairness, bias, explainability, data privacy, and accountability requirements.
196
Key canary metrics to watch?
Business KPIs, error rates, latency, and prediction distribution differences vs baseline.
197
What is a model registry?
Catalogue of model artifacts, metadata, metrics, and lineage for governance and deployment.
198
Core components of MLOps pipeline?
Data ingestion, feature store, training, validation, model registry, CI/CD, and monitoring.
199
Production explainability techniques?
SHAP, LIME, counterfactuals, feature importance dashboards, and rule extraction.
200
What SLA considerations for model APIs?
Latency percentiles, uptime, throughput, error budgets, and scaling guarantees in SLAs.
